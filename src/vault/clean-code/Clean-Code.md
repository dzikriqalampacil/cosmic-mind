# Clean Code

*by Robert C. Martin (Uncle Bob)*

A handbook of agile software craftsmanship. Code is clean if it can be understood easily — by everyone on the team.

> "Clean code reads like well-written prose." — Grady Booch

## Chapters

- [[General Rules]] — Simplicity, DRY, boy scout rule, least surprise
- [[Design Rules]] — Polymorphism, dependency injection, Law of Demeter
- [[Understandability Tips]] — Consistency, explanatory variables, boundary conditions
- [[Names Rules]] — Descriptive, searchable, pronounceable, no encodings
- [[Functions Rules]] — Small, single responsibility, few arguments, no side effects
- [[Comments Rules]] — Self-documenting code, intent, clarification, warnings
- [[Source Code Structure]] — Vertical density, proximity, indentation, line length
- [[Objects and Data Structures]] — Data hiding, single responsibility, static vs instance
- [[Tests]] — F.I.R.S.T. principles, one assert, coverage, readability
- [[Code Smells]] — Rigidity, fragility, immobility, complexity, opacity
- [[Error Handling]] — Exceptions, null safety, context, separation of concerns
